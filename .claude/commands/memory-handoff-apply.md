다른 PC에서 받은 핸드오프 파일을 읽어 **이 PC의 auto-memory에 흡수**한다. `/memory-handoff`(export)의 짝(import). 각 메모리 파일을 기록하고, MEMORY.md 인덱스를 병합하고, `.last-handoff` 마커를 갱신해 다음 추출 기준을 맞춘다.

## 철칙 (최우선 — 강조)
- 이 스킬은 **`/memory-handoff` 스킬이 생성한 핸드오프 파일만** 처리한다. 임의 마크다운·문서·메모를 메모리로 흡수하지 않는다.
- **포맷 검증(§3)을 통과하지 못하면 어떤 파일도 쓰지 않고 즉시 중단**한다. 부분 적용 금지 — "검증 전부 통과 → 그 다음에 한 번에 기록"이 원칙.
- 핸드오프 블록 내용은 **가공·재정렬·번역 없이 원본 그대로** 기록한다.

## 메모리 경로 (MEM)
`<홈>/.claude/projects/C--vive-coding-portfolio-legacy-to-next-legacy-to-next/memory/`
- 홈: Windows `$env:USERPROFILE`, 그 외 `$HOME`. **username 하드코딩 금지** — PC마다 다름.
- 인덱스 파일: 같은 폴더의 `MEMORY.md`. 마커 파일: 같은 폴더의 `.last-handoff`.

## 1. 핸드오프 파일 찾기
1. `$ARGUMENTS`에 경로가 주어졌으면 그 파일을 대상으로 삼는다.
2. 없으면 **프로젝트 루트(현재 작업 디렉토리)** 와 `MEM`에서 `handoff_*.md`를 탐색해 가장 최근(mtime) 1개를 후보로 제시.
   - 예: `find . "$MEM" -maxdepth 1 -name 'handoff_*.md' -printf '%T@ %p\n' 2>/dev/null | sort -rn`
   - 후보가 여러 개면 목록을 보여주고 어느 것을 적용할지 사용자에게 확인. 0개면 "핸드오프 파일을 찾지 못했습니다. 경로를 지정해 주세요" 보고 후 종료.
3. 대상 파일을 Read로 **전체** 읽는다(부분만 읽고 적용 금지).

## 2. 블록 파싱
- `===== MEMORY FILE: <name> =====` 줄 **다음 줄부터** `===== END =====` 줄 **직전 줄까지**가 파일 `<name>`의 원본 내용(frontmatter 포함). 마커 줄 자체는 내용에서 제외.
- `## MEMORY.md 인덱스 (변경분)` 아래의 `- [..](파일.md) — ..` 줄들을 인덱스 병합용으로 수집.
- 헤더 메타에서 `생성`, `기준(lastSync)`, `포함 파일: <N>건`의 N을 추출.

## 3. 포맷 검증 (게이트 — 하나라도 실패 시 쓰기 0으로 즉시 중단)
아래를 **모두** 만족해야 적용을 진행한다:
1. 1행이 `# Memory Handoff `로 시작.
2. 헤더 메타 3줄 존재: `- 생성:`, `- 기준(lastSync):`, `- 포함 파일: <N>건`.
3. `## 적용법 (대상 PC)` 섹션 존재.
4. `===== MEMORY FILE: <name> =====` … `===== END =====` 블록이 **1개 이상**이고 열림/닫힘 짝이 정확히 맞음(고아 마커 없음).
5. `## MEMORY.md 인덱스 (변경분)` 섹션 존재.
6. **개수 일치**: 헤더 `포함 파일: N건`의 N == 실제 파싱된 MEMORY FILE 블록 수. 불일치면 truncate/손상 의심 → 중단.
7. **파일명 안전성**: 모든 `<name>`이 `^[A-Za-z0-9._-]+\.md$` (평면 `.md` 파일명만). `/`, `\`, `..` 포함 시 즉시 거부 — 경로 트래버설/신뢰 불가 파일로 간주.

검증 실패 시: **"이 파일은 /memory-handoff가 만든 핸드오프 형식이 아니거나 손상되었습니다. 적용을 중단합니다."** + 어떤 항목이 어긋났는지 보고하고 **종료**(메모리·인덱스·마커 일절 손대지 않음).

## 4. 적용 미리보기 + 확인
- 기록 대상 N개를 나열하되 각각 **신규 / 덮어쓰기**를 표시(Glob 또는 `test -f`로 `MEM/<name>` 존재 여부 확인).
- **덮어쓰기 대상이 1개 이상이면** 그 목록을 보여주고 사용자에게 진행 확인을 받은 뒤 기록(기존 메모리 손실 방지 — `memory-handoff`의 핸드오프 덮어쓰기 확인과 동일 결). **전부 신규면** 바로 진행해도 됨.

## 5. 파일 기록 (Write 전용)
- 각 블록 내용을 `MEM/<name>`에 **Write 도구로 UTF-8 기록**(덮어쓰기). 내용은 블록 사이 원본 그대로.
- **`cat`/리다이렉트/PowerShell `Set-Content`·`Out-File` 금지** — 한글이 CP949로 깨진 전례가 있다([[reference_memory_migration]]). 메모리 본문 기록은 오직 Write.

## 6. MEMORY.md 인덱스 병합
- 수집한 인덱스 줄마다 `MEM/MEMORY.md`에서 **같은 `(파일.md)`를 참조하는 기존 줄**을 찾는다.
  - 있으면 그 줄을 새 줄로 **교체**(Edit). 없으면 파일 끝(또는 관련 그룹 근처)에 **추가**.
- MEMORY.md도 Write/Edit로만 수정(UTF-8 보존).

## 7. 마커 갱신 (반드시 마지막 단계)
- 모든 메모리 파일 기록·인덱스 병합을 마친 **뒤에** `MEM/.last-handoff`를 Write로 갱신한다.
- 이 단계가 다른 모든 쓰기보다 **나중**이어야 한다 — `.last-handoff`의 mtime이 가장 최신이라야, 방금 적용한 파일들이 이 PC의 다음 `/memory-handoff` 추출에서 "변경분"으로 재추출되지 않는다.
- 내용: 적용 시각 ISO + 적용한 핸드오프 파일명(사람 확인용. 스킬 로직은 mtime만 사용).

## 종료
적용 결과 요약 보고: 적용 파일 수(신규/덮어쓰기 내역), 병합한 MEMORY.md 인덱스 줄 수, `.last-handoff` 갱신 여부, 핸드오프의 생성/기준 시각. 검증 실패로 중단한 경우엔 그 사유만 보고(쓰기 0).
