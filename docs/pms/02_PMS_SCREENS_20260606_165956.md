# 실적 화면 (PMS)

> **상태: 📝 기획 완료 · 구현 예정** — 단일 화면(`PS-01`) 실적 조회.
>
> 본 문서는 **PMS 모듈의 화면 구성**을 정의한다. 집계 도메인·데이터 소스·아키텍처는 `01_PMS_DOMAIN`, 권한 메커니즘은 `docs/admin/03_ACCESS_CONTROL §6.3`을 참조한다(중복 정의 회피).
> **공통 UI 규약**(테마·밀도·폰트·필수 표시, 공용 컴포넌트, 그리드)은 `docs/fms/02_SCREENS 부록 §V1`·`docs/fms/05_NON_FUNCTIONAL §11`을 그대로 준수한다. 컬럼은 표시명(라벨)으로 적고 소스는 `01_PMS_DOMAIN §2·§3`을 인용한다.

## 1. 전체 화면 맵

> PMS 화면은 `PS-` 접두로 식별한다. 라우트는 `(pms)/pms/**`. 운영 화면은 메뉴/버튼 권한으로 노출·실행이 제어된다(`module=PMS`, `docs/admin/03_ACCESS_CONTROL §6.3`).

## 2. PS-01 화면 구성

상단에서 아래로 ① **집계 대상 기준 필터** → ② **조회 조건 바** → ③ **실적 그리드** 의 단일 화면이다.

### 2.1 집계 대상 기준 필터 (PMS 고유)
- **단일 선택 토글**: `운임 입력 건` / `서류 생성` / `세금계산서 발행` / `전표 발행`. 기본값 = `운임 입력 건`.
- 선택값이 집계 모수(대상 행)를 바꾼다(`01_PMS_DOMAIN §2`). **그리드 컬럼 레이아웃은 고정**이며, 선택에 따라 ⓐ 표시되는 B/L 행 집합과 ⓑ 금액 컬럼의 합산 소스(운임/세금/전표 기준=`freight_line`, 서류 기준=`financial_document` 합계)가 전환된다.

### 2.2 조회 조건 바
- FMS·BMS의 조회 필터를 모두 제공한다(§3). 조회는 명시적 트리거(`Search`)에서만 수행하며, `Reset`은 조건·그리드를 비운다(FMS List 정책 답습 — `Search` 트리거 외 자동 조회 금지).
- 일자/거래처/항만은 "종류 선택 + 값" 형태를 따른다(`dateKind`·`partyKind`·`portKind`, §3).

### 2.3 실적 그리드
- **집계 단위 = House B/L**(한 행 = 1 B/L). `01_PMS_DOMAIN §2·§6.1`. 페이지네이션·정렬·합계행(하단 Total)은 공통 그리드 규약을 따른다.
- 코드 컬럼은 코드→이름 동반 표시(§3, `01_PMS_DOMAIN §6.4`).

## 3. 조회 조건 (FMS 전체 + BMS 정산)

> 입력 타입: T=텍스트(like), C=CodeBox(자동완성), E=ENUM 셀렉트, M=다중선택, D=일자, DR=일자범위, K=종류 셀렉터. 대상 컬럼은 House 기준(Master는 대응 컬럼).

### 3.1 공통 (FMS·BMS 교집합)
| 라벨 | 타입 | 대상 |
| --- | --- | --- |
| 업무 구분 (Job Div) | E (SEA/AIR/TRUCK/NON_BL) | `house_bl/master_bl.job_div` |
| 수출입 (Bound) | E (EXP/IMP) | `.bound` |
| 일자 종류 + 범위 | K(ETD/ETA/Perf.) + DR | `etd`/`eta`/`performance_dt` |

## 5. 권한 · 액션
- **라우트 가드**: `MENU_PMS_PERFORMANCE`(예) 권한으로 화면 가드. 없으면 진입 차단.
- **액션 버튼**: `Search` / `Reset` (+ 후속 `엑셀 내보내기` 등은 ActionButton + `admin.button` 정책 시드 동반, `docs/admin/03_ACCESS_CONTROL §6.3·§7`).
- **모듈/사이드바**: `module=PMS` 신규 속성 + 정적 사이드바 등록(`sidebar.tsx` NAV_MODULE + `admin-session` order). 권한 시드와 세트(`docs/admin/03_ACCESS_CONTROL §6.3`).

## 6. 레거시 대비 개선점
- **단일 화면 실적 통합**: 운임·서류·세금계산서·전표 실적을 기준 토글 하나로 한 화면에서 조회한다(별도 리포트 화면 분산 제거).
- **B/L 단위 즉시 집계**: 운임 라인·금융 서류를 House B/L 레벨로 롤업해 코드→이름·B/L 속성과 함께 한눈에 본다.
- **FMS·BMS 필터 통합**: 포워딩 조회 필터와 정산 필터를 한 조회 바에 모아 교차 조회한다.
- **점진 아키텍처**: Phase 1 OLTP 직접 집계로 즉시 가동하고 한계 시 Phase 2 Mart로 전환(`01_PMS_DOMAIN §4`).
