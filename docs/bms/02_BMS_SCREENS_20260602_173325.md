# 정산 화면 (BMS)

> 본 문서는 **BMS 모듈의 화면 목록·구성**을 정의한다. 도메인 모델은 `01_DOMAIN`(엔티티 `B-*`, 컬럼 매핑 §7), 권한 메커니즘·접근 제어는 `docs/admin/03_ACCESS_CONTROL`을 참조한다(중복 정의 회피).
> **공통 UI 규약**(테마·밀도·폰트·필수 표시 등 사용자 환경설정, 공용 컴포넌트)은 `docs/fms/02_SCREENS 부록 §V1` 및 `docs/fms/05_NON_FUNCTIONAL §11`을 그대로 준수한다. 화면 필드는 표시명(라벨)으로 적고, 엔티티 바인딩은 `B-*`로 표기한다(컬럼명은 `01_DOMAIN §7`).

## 1. 전체 화면 맵

> BMS 화면은 `BS-` 접두로 식별한다. 라우트는 `(bms)/bms/**`. 모든 운영 화면은 메뉴/버튼 권한으로 노출·실행이 제어된다(`docs/admin/03_ACCESS_CONTROL`).
> **운임 입력은 BMS 화면이 아니라 FMS B/L Entry의 Freight 탭**에서 이뤄지며 저장은 BMS 테이블(운임 헤더·운임 라인)에 기록된다(연계 명세 §3 "FMS Freight 탭 연계"). **전표(Slip) 발행은 화면 없이 API**로만 수행한다(`01_DOMAIN §6.10`).

| ID | 화면명 | 라우트 | UX 패턴 | 연관 엔티티 |
| --- | --- | --- | --- | --- |
| BS-01 | Invoice List | `/bms/invoice/list` | 마스터-디테일 조회 + 그룹화 | B-03(INVOICE) · B-02 |
| BS-02 | Payment List | `/bms/payment/list` | 마스터-디테일 조회 + 그룹화 | B-03(PAYMENT) · B-02 |
| BS-03 | D/C Note List | `/bms/dc-note/list` | 마스터-디테일 조회 + 그룹화 | B-03(DEBIT+CREDIT) · B-02 |

> BS-01~03은 **공용 리스트 Shell + 종류 config**(FMS S-02~S-05의 공용 Shell+config 방식)로 조립한다. D/C Note는 Debit+Credit 합본.
> **서류 발행·관리 모달**(§3)은 FMS Freight 탭에서 트리거되는 BMS 기능 모달(**발행 + 삭제**, 생성 전용이 아닌 관리 모달)이며 BMS 리스트 화면의 동작이 아니다. **그룹 모달**(§2.2)은 BMS 리스트 화면에서 트리거되는 그룹화 모달이다.

## 2. 공통 UX 패턴

### 2.1 마스터-디테일 조회 (BS-01~03 공통)
- **상단 검색 필터 + 마스터 그리드(금융 서류 B-03)** → 특정 행 선택 시 **하단 디테일 그리드**에 해당 서류에 묶인 **운임 라인(B-02) 세부 정보**가 표시된다.
- **조회 전용**(인라인 편집 아님). FMS List 패턴(검색 필터 + 결과 그리드)에 디테일 그리드를 더한 형태.
- **코드→이름 표시**: 고객·팀·담당자 등은 코드로 저장하고 이름으로 표시한다(ADMIN cross-schema 읽기). **B/L 파생 컬럼**(업무 구분·수출입·House B/L No·ETD·ETA)은 서류 → 운임 → B/L 경유로 표시한다.

### 2.2 그룹화 액션 (BS-01~03)
- **흐름**: 마스터 그리드에서 그룹할 서류 행 선택 → 리스트 내 **그룹 버튼** 클릭 → **그룹 모달** 팝업. 그룹화는 이 3개 조회 화면에서만 수행한다(`group_financial_no`는 서류 List에서만 부여).
- **그룹 모달(좌우 2그리드)**: 진입 시 선택한 서류를 좌/우로 배치한다 — **좌측 = 아직 그룹 안 된 서류 / 우측 = 이미 그룹된 서류**. 좌→우 이동 = 그룹 부여, **우→좌 이동 = 그룹 해제**.

### 2.3 서류 발행 (모달, FMS Freight 탭 트리거)
- 운임 라인을 금융 서류로 묶어 발행하는 작업은 **FMS Freight 탭의 발행 버튼 → 서류 발행·관리 모달**에서 수행한다(§3). BMS 리스트 화면에서는 발행하지 않는다(리스트는 조회 + 그룹화).

### 2.4 권한 기반 렌더
- 화면(라우트)·액션 버튼은 `MENU_*`/`BTN_*` 권한으로 가드·렌더한다(`docs/admin/03_ACCESS_CONTROL`).

## 3. 화면별 상세

### BS-01 / BS-02 / BS-03: 금융 서류 조회 리스트 (공용 Shell)
- **구조**: 마스터(서류)-디테일(운임 라인) 2그리드 조회(§2.1). 라우트별 `document_type` 고정 — Invoice / Payment / Debit+Credit(D/C Note).
- **검색 필터**: 업무 구분 · 수출입 구분 · 일자 필터(서류 일자 / ETD / ETA / Performance Date 범위) · 서류 Status · Customer · Document No · Team · Operator.
- **마스터 그리드 컬럼**(금융 서류 + 파생): Status · 서류 번호 · 그룹 번호 · 고객 코드 · 고객 명 · 서류 일자 · 실적 일자 · 업무 구분 · 수출입 구분 · House B/L No · Team Name · Operator Name · ETD · ETA · USD Amount · Local Amount · Local VAT.
  - 출처: 고객 명 / Team Name / Operator Name = 코드→이름(ADMIN). 업무 구분·수출입·House B/L No·ETD·ETA = 링크된 B/L(FMS). USD Amount·Local Amount·Local VAT = 서류 합계 스냅샷(B-03).
- **디테일 그리드 컬럼**(선택 서류에 묶인 운임 라인 B-02): 고객 · 고객 명 · 운임 · 운임명 · 통화 · 환율 · Per · 단위 수량 · 단위 가격 · 발행 금액(단위 수량 × 단위 가격) · 로컬 금액 · 세금 구분(영세/면세/과세) · 세액 · 총 금액(로컬 금액 + 세액) · 전표 번호 · Tax 번호.
  - 발행 금액 = 정산 통화 금액(B-02). 로컬 금액·세액·총 금액은 로컬(KRW) 기준. 발행 금액·총 금액은 계산 표시 컬럼.
- **액션**: 조회(Search / Reset) · 그룹화(§2.2). **세금계산서 발급·전표 발행은 화면 액션이 아니라 API**로 수행한다(전표는 `01_DOMAIN §6.10`).
- **연관 엔티티**: B-03 · B-02 (+ ADMIN 이름 해석, FMS B/L 파생).

### 서류 발행·관리 모달 (FMS Freight 탭에서 트리거)
> 생성 전용이 아닌 **관리 모달**: 발행(생성) + 삭제.
- **트리거**: FMS B/L Entry(S-02 House / S-04 Master) Freight 탭의 **각 Selling / Buying 그리드 우측 상단 발행 버튼**. 운임은 **B/L 저장 후**에만 서류로 묶을 수 있다(`01_DOMAIN §6.17`).
- **대상·종류**: 모달은 해당 그리드의 운임 라인 **전체**를 대상으로 띄운다. **서류 종류는 진입 시점에 이미 결정**되어 있다(라인의 종류 = 고객 종류 × 매출/매입 자동 산정, `01_DOMAIN §6.16`). **한 모달 = 한 종류 서류**.
- **라인 선택**: **체크박스 다중 선택**. **동일 고객만** 묶을 수 있으며 다른 고객이 섞이면 **토스트 경고 + 차단**(§6.14).
- **헤더 입력(서류 단위)**: 서류 1 : 운임 N이므로 **헤더에서 서류 일자 · 실적 일자 · 팀 · 오퍼레이터를 일괄 입력**한다. 기본값 — 서류 일자 = 오늘(수정 가능), 실적 일자 = 헤더 일괄 입력(서류 기준 부여 → 라인 전파 §6.15), 팀 = 로그인 팀(수정 가능), 오퍼레이터 = 로그인 사용자(대무 시 변경 가능). 고객·합계는 자동. **`group_financial_no`는 모달에서 설정하지 않는다**(서류 List 그룹 모달에서만, §2.2).
- **라인 영역**: 선택한 라인 정보가 **집계 없이 그대로** 들어오며 발행 전 라인 값 수정 가능(종류 고정).
- **발행 처리(자동)**: 서류 번호 채번(§6.11) · 운임 라인 연결 · 실적 일자 라인 전파(§6.15) · 합계 스냅샷(§6.13). **스냅샷 후 묶인 운임 라인은 값 변경 불가**(§6.17).
- **발행 후 거동**: 모달은 **닫히지 않고 잔류**해 발행 결과를 확인한다. **모달을 나갈 때 Freight 탭 Account Documents 그리드만 재조회**되어 반영된다.
- **삭제(관리)**: **모달 내 삭제 버튼**으로 서류를 삭제한다. 발행 내용 수정은 **삭제 후 재발행**으로만 한다(스냅샷 정책 고수, §6.17). 서류에 묶인 운임 라인은 **묶인 서류에서 풀어낸(삭제) 뒤에만** 행 삭제 가능.

### FMS Freight 탭 연계 (S-02 House / S-04 Master)
> FMS B/L Entry의 Freight 탭이 본 모듈(운임 헤더 B-01 · 운임 라인 B-02)에 바인딩된다. **FMS SCREENS의 Freight 탭 절(§240 부근)은 아래 바인딩으로 보정 필요** — 보정안은 본 작업 산출과 함께 별도 제공.
- **Rate Header → 운임 헤더(B-01)**: 실화주 · 선사 · 정산처 + **환율 기준(통화 · 일자)**.
- **Selling/Buying 그리드 → 운임 라인(B-02)**: 매출/매입별 분리 표시(저장은 단일 엔티티).
- **Account Documents → 해당 B/L에 묶인 금융 서류(B-03) 그리드**(조회 표시). **체크박스로 서류 다중 선택 후 삭제** 지원. 발행·관리 모달을 나갈 때 이 그리드만 재조회되어 반영된다.
- **각 Selling/Buying 그리드 우측 상단 서류 발행 버튼 → 서류 발행·관리 모달**(위). 발행(서류로 묶기)은 **B/L 저장 후** 가능하다(`01_DOMAIN §6.17`).
- **행 삭제 보호**: 서류에 묶인 운임 라인은 행 삭제 불가 — 묶인 서류에서 풀어낸 뒤(서류 삭제) 삭제한다. 발행(스냅샷)된 라인은 값 변경 불가(`01_DOMAIN §6.17`).
- **Remark 영역 제거**(설계 미스).
- 저장은 BMS 테이블(쓰기 소유권 BMS). Freight 탭은 **운임 입력 전용**이다.

## 4. 레거시 대비 개선점
- **운임 입력의 B/L 맥락 통합**: 운임/정산 Raw 입력을 B/L Entry Freight 탭에 통합해 B/L 맥락에서 바로 입력·발행한다.
- **마스터-디테일 조회 일원화**: 서류와 그에 묶인 운임을 한 화면에서 즉시 확인(서류 그리드 → 운임 라인 디테일).
- **조회 한 화면 완결**: 코드→이름 해석과 B/L 파생 컬럼으로 별도 화면 이동 없이 조회를 마친다.
