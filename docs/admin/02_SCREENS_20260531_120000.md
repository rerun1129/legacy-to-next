# 관리 화면 (ADMIN)

> 본 문서는 **ADMIN 모듈의 화면 목록·구성**을 정의한다. 도메인 모델은 `01_DOMAIN`(엔티티 `A-*`), 권한 메커니즘·접근 제어는 `03_ACCESS_CONTROL`을 참조한다(중복 정의 회피).
> **공통 UI 규약**(테마·밀도·폰트·필수 표시 등 사용자 환경설정, 공용 컴포넌트)은 `docs/fms/02_SCREENS 부록 §V1` 및 `docs/fms/05_NON_FUNCTIONAL §11`을 그대로 준수한다 — 본 문서는 ADMIN 고유 화면·UX 패턴만 명세한다.

## 2. 공통 UX 패턴

> ADMIN 운영 화면은 모달 위주의 레거시 CRUD를 벗어나 **인라인 그리드** 중심으로 통일됐다. 아래 4개 패턴으로 분류한다. 화면별 상세(§3)는 이 패턴을 참조한다.

### 2.1 P1 — 인라인 그리드 (Reset / Search / Save)

운영 마스터 관리의 기본 패턴. 한 화면에서 조회·편집·저장을 인라인으로 처리한다.

- **툴바 버튼**: `Reset` / `Search` / `Save` (섹션마다 `Save` 1개)
  - **Reset = 그리드 비우기** — 필터·그리드 행을 모두 비운다. **조회를 트리거하지 않는다**(활성 필터 null + 폼 rows=[]). 캐시 invalidate를 호출하지 않는다.
  - **Search = 명시 조회** — 현재 필터로 조회하고 결과를 그리드에 로드. List 갱신은 Search 트리거에서만 일어난다.
  - **Save = 일괄 저장** — 그리드의 추가/수정 변경분을 한 번에 전송. payload는 **creates / updates만** 포함한다(삭제는 deactivate, §2.5).
- **행 편집**: 셀 직접 입력, 행 추가(Add)·제거(Remove). 클립보드 **붙여넣기(paste)** 지원(Tab/Enter 구분 → 셀 매핑).
- **변경 수집**: 신규/수정/삭제 행을 분류해 저장 payload로 구성. RHF `useFieldArray` 사용 시 행 식별은 `entityId`로 분리한다(RHF가 `id`를 UUID로 덮어쓰는 문제 회피).
- **자동완성**: 코드/참조 필드(예: User의 팀)는 CodeBox 자동완성으로 입력(`docs/fms/02_SCREENS` 부록의 CodeBox 규약 공유).

### 2.2 P2 — 마스터-디테일 (Code List, AS-09)

좌측 Code Master 인라인 그리드 / 우측 선택 Master의 Code Detail 그리드. Master 행 선택 시 해당 Detail이 로드된다. Detail 미저장 변경(dirty) 상태에서 Master를 전환하면 저장 확인을 거친다.

### 2.3 P3 — 트리 + 정책 패널 (Menu AS-05 / Button AS-06)

- **트리**: 메뉴 계층(`parent_id` self-ref) 또는 메뉴→버튼 트리(expand/collapse).
- **정책 패널**: 선택한 메뉴/버튼의 정책(`attribute_key`·`required_value`, `01_DOMAIN A-09·A-11`)을 표시·편집. 정책 평가 규칙은 `03_ACCESS_CONTROL`.

### 2.4 P4 — 멀티셀렉트 섹션 (Permission Preset, AS-08)

프리셋 선택 시 하단에 해당 프리셋이 포함하는 **속성값(A-04) 멀티셀렉트** 섹션이 열린다. 선택 변경을 저장하면 프리셋 보유 사용자 전원에 즉시 반영된다(live binding, `01_DOMAIN A-05`). 다중 값 붙여넣기 지원.

### 2.5 식별 코드 삭제 = deactivate-only

`menu_code`·`button_code` 등 식별 코드 행은 **하드 삭제하지 않고 `active` 토글로만 비활성화**한다(`01_DOMAIN §6.6`). 저장 payload는 creates/updates만 다룬다.

### 2.6 권한 기반 렌더 (AdminGuard / ActionButton)

- **AdminGuard**: 화면(라우트)을 `MENU_*` 권한으로 가드. 권한 없으면 진입 차단.
- **ActionButton**: 액션 버튼을 `BTN_*` 권한으로 렌더. **권한이 없으면 렌더하지 않는다(return null)** — 따라서 신규 ActionButton 도입 시 `admin.button` + 정책 시드가 동반되어야 화면에 보인다.
- **버튼 라벨**: ActionButton은 `icon` prop으로 식별하고 **표시 라벨은 DB(`button.label`)에서 자동 주입**된다(children은 fallback).
- **탭 제목 출처**: 화면 탭 제목은 프론트의 라우트 라벨 맵(`use-tabs`)에서 결정되며 DB `menu.label`이 아니다(리다이렉트형 탭은 재로그인 시 반영).

## 3. 화면별 상세

### AS-01: Admin Home
- **구조**: 관리 진입 대시보드 셸. 주요 카운트·바로가기 중심(정적). 동적 위젯은 후속.

### AS-02: User List
- **핵심 기능**: 사용자 검색(username/email/active) + 인라인 편집. 팀 배정은 **Team CodeBox 자동완성**(team_id 저장, team_code 표시). 속성/프리셋 배정 연계는 `03_ACCESS_CONTROL`.
- **연관 엔티티**: A-01 Admin User, A-02 Team.

### AS-03: Customer List
- **핵심 기능**: 거래처 검색(code/type/name) + 인라인 편집. FMS 당사자(Party)의 원천 마스터.
- **연관 엔티티**: A-22 Customer.

### AS-04: Notice List
- **핵심 기능**: 공지 그리드(pinned/published_at/expires_at) + **신규/수정 모달**. 게시·만료 기간으로 노출 제어. 대시보드 MJ Notice 위젯의 원천.
- **비고**: 모달 전용 조회는 재오픈 시 최신성을 위해 `gcTime:0`로 운용(unmount 후 stale 캐시 회피).
- **연관 엔티티**: A-23 Notice.

### AS-05 / AS-06: Menu / Button
- **핵심 기능**: P3 트리 + 정책 패널. 메뉴 트리(self-ref)·메뉴별 버튼 카탈로그 관리 및 각 항목의 접근 정책 편집. 정책 0개 부모 메뉴의 over-expose는 Service 후처리("자식 0개 root 제거")로 방지(`01_DOMAIN §6.5`).
- **연관 엔티티**: A-08·A-09 / A-10·A-11.

### AS-07: Attribute
- **핵심 기능**: 상단 Attribute Definition(A-03) 인라인 그리드 → 선택 시 하위 Attribute Value(A-04) 섹션(value/label/sort/active). ENUM 타입 속성의 허용 값을 관리.
- **연관 엔티티**: A-03·A-04.

### AS-08: Permission Preset
- **핵심 기능**: 프리셋(A-05) 인라인 그리드 + P4 멀티셀렉트(포함 속성값). 사용자↔프리셋 배정(A-07)과 함께 권한 운영의 핵심 경로. 프리셋 코드는 `PRESET_*` 강제.
- **연관 엔티티**: A-05·A-06·A-07.

### AS-09: Code List
- **핵심 기능**: P2 마스터-디테일. 공통 코드 그룹(A-12)과 코드 값(A-13) 인라인 관리.
- **연관 엔티티**: A-12·A-13.

### AS-10~AS-17: 특화 코드 (Carrier/Country/Currency/ExchangeRate/Freight/HsCode/PackageUnit/Port)
- **핵심 기능**: P1 인라인 그리드 공통. 컬럼은 각 엔티티 고유 필드(`01_DOMAIN §2.2 A-14~A-21`). 코드 자동완성·prefix 우선 정렬 적용.
- **연관 엔티티**: A-14~A-21.

## 4. 레거시 대비 개선점 (공통)
- **모달 CRUD → 인라인 그리드**: 레거시의 행별 모달 입력을 한 화면 인라인 편집 + 일괄 저장으로 전환해 대량 코드 관리 효율을 높였다.
- **권한 일원화**: 화면·버튼 노출을 ABAC 정책(`03_ACCESS_CONTROL`)으로 통일해 하드코딩 분기를 제거.
- **붙여넣기 입력**: 외부 시트에서 복사한 코드 표를 그대로 붙여넣어 초기 적재 비용을 절감.
